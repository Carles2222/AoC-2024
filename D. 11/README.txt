El primer código (Parte 1) simula 25 parpadeos de las piedras, siguiendo las reglas dadas. Usa listas para manejar el estado actual de las piedras, y en cada parpadeo genera un nuevo estado aplicando las transformaciones a cada piedra.
El segundo código (Parte 2) realiza la misma simulación pero para 75 parpadeos, utilizando un enfoque más eficiente con diccionarios. En lugar de rastrear cada piedra individualmente, agrupa las piedras por su valor y las procesa en lotes, reduciendo el costo computacional. Ambos códigos aplican las reglas de transformación de las piedras basándose en su número y el caso que les corresponda.
